# Sumário

* Monolito procedural
  * View com Swing. Formulário
  * Script único para toda a aplicação
  * Persistência dos dados em arquivos de texto
* Uma segunda aplicação monolítica procedural
  * O problema da sincronização de dados 
* Cliente servidor gordo
  * Conexão JDBC
  * Arquivo para isolar as queries
* Cliente servidor magro
  * Procedures no banco
* Arquitetura de três camadas
  * Interface gráfica swing
  * Aplicação com as regras de negócio
  * Banco de dados como repositório
* Arquitetura web
  * O protocolo HTTP
  * Respostas HTML
  * O servidor de aplicação
  * O front-controller
  * A inversão de dependências e o pool de recursos
* Arquitetura MVC
  * O domínio da aplicação
  * Bindind da interface gráfica com o modelo
  * Mapeamento ORM do modelo
* Arquitetura de serviços
  * Web services que se comunicam sem a intervenção humana
  * XML e XSD
  * JSON
* Arquitetura de microsserviços
  * Bases de dados separadas
  * Comunicação por filas de mensageria
  * Bibliotecas externas e o compartilhamento de modelos
